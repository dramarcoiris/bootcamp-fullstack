¿Por qué conviene escribir HTML semántico?

Cuando comenzamos a aprender desarrollo web, una de las primeras cosas que aprendemos es el uso de las etiquetas HTML para estructurar el contenido. Al principio puede abrumar, pero es necesario conocer la función de cada una para escribir un código limpio y estructurado.

¿Qué significa HTML semántico?

Se refiere al uso de etiquetas que describen su propósito y el tipo de contenido que contienen. Usamos por ejemplo la etiqueta <header> para la cabecera de una página, <nav> para la navegación, <section> para secciones de contenido, <article> para un artículo, y <footer> para el pie de página.

El uso de etiquetas semánticas mejora la estructura del documento y facilita la comprensión del contenido tanto para los desarrolladores como para los motores de búsqueda.

Guías de buenas prácticas en desarrollo web

¿Cuáles son las ventajas?

Una estructura clara permite que otras personas entiendan mejor el código. Además, muchas guías de desarrollo repiten la misma idea: la estructura del contenido es tan importante como su diseño visual.

Ejemplo de código semántico

Un ejemplo básico de HTML semántico sería estructurar una página utilizando etiquetas con significado en lugar de usar solo <div>.


            <header>
                <h1>Mi página web</h1>
            </header>
    
            <main>
                <p>Contenido principal</p>
            </main>

Se puede aprender más sobre HTML semántico en la documentación de MDN Web Docs.
